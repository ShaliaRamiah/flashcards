What is OOP?

A programming paradigm that revolves around the concept of "objects."

Encapsulation?

Encapsulation is the bundling of data (attributes or properties) and the methods (functions) that operate on that data, restricting access to some of the object's components.

Example: In a class representing a Car, you may have private attributes like engineStatus and fuelLevel, and public methods like startEngine() and refuel() to control and interact with these attributes.

Inheritance?

Inheritance is the mechanism by which one class can inherit properties (fields and methods) from another class. The class that inherits is called a subclass, and the class it inherits from is called a superclass or parent class.

Example: If you have a Vehicle class, you can create subclasses like Car and Motorcycle that inherit attributes and methods from Vehicle, but may have additional specific features.

Polymorphism?

Polymorphism means the ability of a class to take on multiple forms. In OOP, it refers to the ability of a class to provide different implementations of methods that are inherited from its parent class or defined in an interface.

Example: Inheritance combined with polymorphism allows you to have a list of Vehicle objects that can hold instances of both Car and Motorcycle, and still call a common method like drive(), but the actual behavior may differ.

Abstraction?

Abstraction is the process of hiding the implementation details of an object and exposing only the relevant features or properties. It focuses on what an object does rather than how it does it.

Example: If you're using a library or framework, you don't need to know how each function is implemented; you only need to understand its purpose and how to use it.

A class is a blueprint for creating objects. Classes are used for creating instances of objects, while interfaces are used for defining the structure that objects should have.

What is a Constructor?

A constructor is a special method within a class that is responsible for initializing the object of that class. It is called when an instance of the class is created or "constructed."

What is the "static" keyword?

The "static" keyword is useful when you want to create variables or methods that are not tied to specific instances but rather pertain to the entire class.

What is the "final" keyword?

The "final" keyword is used to declare entities that cannot be changed or modified once they are assigned a value.

What is a class?

A class is a blueprint or template that defines the structure and behavior of an object. It serves as a blueprint for creating individual objects.

When do you use '==' and '==='?

==: compares the value of the variables. ===: compares the type and value of the variables.
